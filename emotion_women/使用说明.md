# 情感女性公众号 · 使用说明

一套基于 Claude Code 的全自动情感/女性向公众号内容生产系统。Python 脚本负责调度和注入动态 Prompt，Claude Code 主进程扮演**主编**搜热点、筛选题、去重、派活，`emotion-writer` 子智能体扮演**写手**做研究并成稿。

> 想快速上手直接看「一分钟上手」；想搞清原理和发布配置往下翻。

---

## 一分钟上手

```bash
cd emotion_women

# 立即生成 1 篇（带详细日志，推荐首次运行用这个观察全过程）
python3 daily_emotion_women.py --now --count 1 --verbose
```

跑完后草稿在 `articles/` 下，文件名形如 `20260702_2230_sunk-cost-in-love.md`。默认**只生成本地草稿，不碰公众号**。

## 运行模式

脚本有两个维度的模式：**触发方式**（立即 / 定时）× **是否发布**（本地草稿 / 推送草稿箱）。默认不发布，只有显式加 `--publish` 才会调用公众号发布工具。

### 1. 立即生成（手动触发）

```bash
# 生成 1 篇
python3 daily_emotion_women.py --now --count 1

# 生成 3 篇，带详细日志
python3 daily_emotion_women.py --now --count 3 --verbose
```

`--now` 表示立即执行一次就退出，不进入定时循环。

### 2. 发布到公众号草稿箱

```bash
# 生成 1 篇，并推送到微信公众号草稿箱
python3 daily_emotion_women.py --now --count 1 --publish --verbose

# 生成 3 篇，并逐篇推送到草稿箱
python3 daily_emotion_women.py --now --count 3 --publish
```

加上 `--publish` 后，脚本会加载 `emotion_women/.mcp.json`，通过 `wenyan-mcp` 把成稿排版后推送到微信公众号**草稿箱**。它不会直接群发，仍需你在微信公众平台后台人工审核。

成功后去这里检查：微信公众平台 → 内容与互动 → 草稿箱。

### 3. 定时调度（后台常驻）

```bash
# 每天北京时间 09:00 和 21:00 各生成 3 篇
python3 daily_emotion_women.py --time 09:00 21:00 --count 3

# 每天北京时间 21:00 生成 1 篇，并推送到草稿箱
python3 daily_emotion_women.py --time 21:00 --count 1 --publish

# 后台守护运行
nohup python3 daily_emotion_women.py --time 09:00 21:00 --count 3 > run.log 2>&1 &
```

- `--time` 可跟多个时间点，格式 `HH:MM`（24 小时制，北京时间）。
- 定时模式是前台阻塞循环（每 60 秒轮询一次），按 `Ctrl+C` 退出。
- 需要 `schedule` 库：`pip install schedule`。

> 💡 情感类内容建议早晚各发一轮（如 09:00 + 21:00），晚间阅读高峰更适合情感话题。

### 3. 在 Claude Code 中交互使用

```bash
cd emotion_women
claude
```

进入目录后 Claude Code 自动加载 `CLAUDE.md` 配置，你可以直接对话：

```
写一篇关于"分手后为什么总是女生在反思自己"的文章
```

```
根据最近微博热搜写一篇情感文章
```

```
帮我写一篇关于讨好型人格在恋爱中有多累的文章
```

Claude 会调用 `emotion-writer` agent 完成搜索、写作和发布。

## 命令行参数

| 参数 | 说明 |
| --- | --- |
| `--now` | 立即执行一次，不等待定时调度 |
| `--count N` | 每次执行生成 N 篇文章（默认 3） |
| `--time HH:MM [HH:MM ...]` | 设定每日执行的北京时间，可多个 |
| `-v` / `--verbose` | 输出详细日志，能看到主编和写手的完整调度过程 |
| `--publish` | 成稿后通过 `wenyan-mcp` 发布到微信公众号草稿箱（需先配置 `.mcp.json`） |

## 工作流程

```
Python 调度层
    └─► Claude Code 主进程（主编 / Agent Manager）
            ├─ 1. WebSearch 搜最近 24h 的情感/女性话题热点
            ├─ 2. 筛选选题 + 去重（检查 articles/ 已有文章）
            ├─ 3. 分派 emotion-writer 子智能体（一个选题一个 agent）
            │        └─► 写手：深度研究 → 下载配图 → 写 Markdown → 保存
            ├─ 4.（--publish 时）调 wenyan-mcp 推送到草稿箱
            └─ 5. 汇报每篇标题、路径、摘要、发布状态
```

1. **调度触发** — 按 `--now` 或 `--time` 启动。
2. **动态 Prompt** — 脚本注入当前北京时间、搜索方向和选题标准。
3. **主编执行** — 搜热点 → 去重 → 分派写手。
4. **写手撰写** — 多源研究 → 下载配图 → 按风格规范写文章 → 存到 `articles/`。
5. **发布** — 仅在 `--publish` 模式下，通过 wenyan-mcp 推到草稿箱，人工审核后群发。

## 发布到公众号（可选）

默认 `articles/` 只是本地草稿。要推送到微信公众号草稿箱：

### 1. 准备公众号开发配置

在微信公众平台获取情感女性公众号的开发凭证：

- 路径：微信公众平台 → 开发 → 基本配置
- 需要：`AppID` 和 `AppSecret`
- 确认公众号已完成必要的开发配置，AppSecret 可用

### 2. 构建 wenyan-mcp

```bash
git clone https://github.com/floodsung/wenyan-mcp.git
cd wenyan-mcp && npm install && npm run build
```

构建后确认存在：

```bash
/你的实际路径/wenyan-mcp/dist/index.js
```

### 3. 配置 .mcp.json

先从模板复制一份真实配置：

```bash
cd emotion_women
cp .mcp.example.json .mcp.json
```

然后编辑 `emotion_women/.mcp.json`：

```json
{
  "mcpServers": {
    "wenyan-mcp": {
      "type": "stdio",
      "command": "node",
      "args": [
        "/你的实际路径/wenyan-mcp/dist/index.js"
      ],
      "env": {
        "WECHAT_APP_ID": "你的情感公众号AppID",
        "WECHAT_APP_SECRET": "你的情感公众号AppSecret"
      }
    }
  }
}
```

`.mcp.example.json` 可以提交到 Git，因为它只放占位符；`.mcp.json` 位于 `emotion_women/` 目录下，里面是本频道专用的公众号凭证。项目根目录 `.gitignore` 已忽略 `**/.mcp.json`，不要把它提交到 Git。

### 4. 执行发布命令

进入情感号目录：

```bash
cd emotion_women
```

首次建议只发 1 篇并开启详细日志：

```bash
python3 daily_emotion_women.py --now --count 1 --publish --verbose
```

稳定后可以批量：

```bash
python3 daily_emotion_women.py --now --count 3 --publish
```

定时发布到草稿箱：

```bash
python3 daily_emotion_women.py --time 18:03 --count 1 --publish
```

### 5. 判断是否成功

命令输出里看到类似下面的信息，说明已经进草稿箱：

```text
发布到公众号草稿箱
media_id: ...
```

然后到微信公众平台后台检查：内容与互动 → 草稿箱。发布前建议人工检查标题、封面、配图、错别字和敏感表达。

如果命令生成了 `articles/*.md`，但没有 `media_id`，说明文章已本地生成，公众号发布环节失败。优先检查 `.mcp.json` 路径、AppID/AppSecret、`wenyan-mcp/dist/index.js` 是否存在。

如果失败原因是：

```text
微信公众号 API IP 白名单限制
```

按下面处理：

1. 复制日志里提示的当前服务器 IP。例如本次日志提示的是 `49.65.67.21`。
2. 登录微信公众平台：设置与开发 → 基本配置 → IP 白名单。
3. 把该 IP 加入白名单并保存。
4. 不要重新生成文章，直接重试发布已有 Markdown：

```bash
python3 publish_existing_article.py articles/20260703_2200_roommate-marriage.md --verbose
```

如果网络环境变化，出口 IP 可能也会变化。以后遇到同类报错，以当次日志里提示的 IP 为准。

### 6. 排版主题

情感类文章当前默认使用 `purple` 主题；如果本地 `wenyan-mcp` 支持 `agentera-rose`，也可以再切换为玫瑰金风格。其他可选：

| theme_id | 风格 | 适用场景 |
| --- | --- | --- |
| `purple` | 紫色简洁 | 当前可用的情感号默认主题 |
| `agentera-rose` | 玫瑰金优雅 | 情感故事、女性话题（如果本地 wenyan-mcp 支持） |
| `agentera-galaxy` | 深蓝星系 | 深度思考、心理分析 |
| `agentera-mint` | 薄荷清新 | 治愈系、轻松话题 |
| `agentera-orange` | 橙金活力 | 正能量、成长类 |

> ⚠️ `.mcp.json` 含微信 AppSecret，**切勿提交到版本库**。建议加入 `.gitignore`。

## 发布到小红书（可选）

小红书没有官方开放的发文 API，也**没有草稿箱**。这里借助开源项目
[`xiaohongshu-mcp`](https://github.com/xpzouying/xiaohongshu-mcp)——它用浏览器自动化模拟真人操作发布。

> ⚠️ 与公众号最大的不同：小红书 `publish_content` 是**直接发布上线**，不是进草稿箱。
> 因此本项目的 `publish_to_xiaohongshu.py` **默认以「仅自己可见」发布**，供你先在手机上确认效果，
> 满意后再用 `--visibility 公开可见` 正式公开。

### 1. 部署并启动 xiaohongshu-mcp

已在 `/Users/xiao/codeworkspace/自媒体/xiaohongshu-mcp/` 放好 `docker-compose.yml`。

```bash
cd /Users/xiao/codeworkspace/自媒体/xiaohongshu-mcp
docker compose up -d          # 首次会拉取镜像，稍慢
docker compose logs -f        # 看日志，等待服务就绪
```

服务地址：`http://localhost:18060/mcp`。验证：

```bash
curl -s -X POST http://localhost:18060/mcp \
  -H "Content-Type: application/json" \
  -H "Accept: application/json, text/event-stream" \
  -d '{"jsonrpc":"2.0","method":"initialize","params":{"protocolVersion":"2024-11-05","capabilities":{},"clientInfo":{"name":"c","version":"1"}},"id":1}'
```

### 2. 首次登录（手机扫码，只做一次）

在 Claude Code 里对话即可（`emotion_women/.mcp.json` 已注册 `xiaohongshu-mcp`）：

```
帮我检查小红书登录状态
```

若未登录，让它调用 `get_login_qrcode` 返回二维码，用**小红书 App 扫码**登录。
cookie 会存到 `xiaohongshu-mcp/data/`，之后复用，无需重复扫码。

### 3. 把文章改写成小红书笔记

公众号长文（1200-1800 字）不能直接发小红书。硬约束：

- **标题 ≤ 20 字**
- **正文 ≤ 1000 字**
- **至少 1 张图片**（可用文章里的 Unsplash 链接，或 `images/` 下本地图的绝对路径）

改写后存成笔记文件 `articles/xhs/<slug>.xhs.md`，格式：

```markdown
---
title: 20字以内的标题
images:
  - https://images.unsplash.com/photo-xxxx?w=1080&q=80
tags:
  - 情绪疗愈
  - 女性成长
visibility: 仅自己可见   # 缺省时用命令行 --visibility
---
正文（≤1000字，可含 emoji、换行、话题引导）...
```

参考示例：`articles/xhs/20260705_1502_solo-recovery.xhs.md`。

### 4. 发布

```bash
# 验证：仅自己可见（默认）。发布前会打印笔记内容并要求人工确认。
python publish_to_xiaohongshu.py articles/xhs/20260705_1502_solo-recovery.xhs.md -v

# 手机上确认没问题后，正式公开
python publish_to_xiaohongshu.py articles/xhs/20260705_1502_solo-recovery.xhs.md --visibility 公开可见
```

参数：`--visibility {仅自己可见,公开可见,仅互关好友可见}`、`--yes`（跳过确认）、
`--url`（自定义 MCP 地址，默认 `http://localhost:18060/mcp`）。

### 常见问题

- **连不上 MCP**：确认 `docker compose ps` 里容器在运行，`docker compose logs` 无报错。
- **提示未登录**：重新走第 2 步扫码；或让 Claude 调用 `delete_cookies` 后重新登录。
- **发布报违禁词/限流**：先把正文里的敏感词删掉，小红书对营销/极端表述较敏感。

## 内容定位与选题方向

### 目标读者

22-35岁都市女性，在感情中经历过迷茫、受伤，正在寻找自我价值和情感智慧。

### 四大内容方向

| 方向 | 举例话题 |
| --- | --- |
| **两性情感** | 分手心理重建、恋爱中的控制欲、出轨分析、沉没成本 |
| **女性成长** | 经济独立、讨好型人格自救、职场性别困境、30岁焦虑 |
| **情绪疗愈** | 内耗识别、断联适应期、走出PUA阴影、允许不完美 |
| **热点切入** | 明星情感事件解读、热播剧两性分析、社会新闻女性视角 |

### 写作风格

- **像闺蜜深夜聊天**，不是专家上课
- **故事驱动**：每篇至少1个完整场景
- **观点犀利**：不和稀泥，有清晰价值判断
- **金句穿插**：每300-400字有一个值得截图的句子
- **反AI检测**：禁空洞鸡汤、禁说教口吻、禁过渡词堆砌

## 常见问题

**Q：跑完提示成功，但文章像AI写的鸡汤？**
A：检查 `emotion-writer.md` 中的反AI规则是否生效。如果写手绕过了规则，可以在主编 Prompt 中加强约束，或在 `CLAUDE.md` 中追加"质量否决"逻辑。也可以尝试换用 `opus` 模型（在 agent 定义的 frontmatter `model: opus` 修改），观点输出会更有深度。

**Q：配图下载失败或图片是广告？**
A：agent 会用 Read 工具验证每张图片。如果网络受限导致下载失败，可以手动在 `images/` 中放几张通用的意境图（女性独处、城市夜景等），然后在写作时引用。

**Q：想换成其他公众号的凭证？**
A：只需修改 `.mcp.json` 中的 `WECHAT_APP_ID` 和 `WECHAT_APP_SECRET`，指向你的情感类公众号即可。

**Q：`--time` 报错说缺 `schedule`？**
A：`pip install schedule` 后重试。

**Q：找不到 `claude` 命令？**
A：先安装并登录 Claude Code CLI，确认 `claude --version` 可用。

**Q：如何调整每日选题数量？**
A：`--count` 参数控制。情感类建议每天 2-3 篇为宜，保证质量优先。
