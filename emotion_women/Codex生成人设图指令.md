# Codex 生成人设图指令

## 使用方法

进入 Codex，在项目目录中直接粘贴下面的指令。

只需要修改“本次要求”部分，身份母图和造型参考不要更换。

## 单张人物图

~~~text
使用 imagegen 技能生成人设图。

唯一身份参考：
emotion_women/images/persona/master/persona_master_glam_v3.png

造型参考：
emotion_women/images/persona/style/persona_style_sensual_v1.jpg

身份母图决定她是谁，造型参考只控制穿搭和成熟妩媚的气质。

必须保持同一个成年女性的脸型、眼睛形状和间距、鼻子、嘴唇、
下颌、年龄、肤色、发际线和黑色齐肩头发。

本次要求：
场景：雨夜便利店门口
造型：灰色修身风衣、白色内搭
表情：疲惫但平静
姿势：一手拿透明雨伞，一手看手机
视线：低头看手机
镜头：中景，眼平机位，35mm，自然抓拍
光线：便利店暖光和雨夜冷光
正在发生的动作：她刚从便利店出来，低头确认打车信息
物体接触：右手握伞柄，左手拇指划过手机屏幕
生活痕迹：门口地垫有雨水，塑料伞面挂着水珠
用途：公众号正文配图

图片风格：
高度写实的生活摄影，精致日常妆容，眼睛和人脸锐利对焦，真实毛孔、细小皮肤纹理、自然碎发、
真实布料褶皱、合理光影和透视，不使用磨皮和网红滤镜。
画面必须像人物正在过日子，不是换了背景继续摆拍。

避免：
塑料皮肤、蜡像脸、五官漂移、年龄变化、夸张身材、
错误手指、融合牙齿、异瞳、乱码、品牌、Logo、水印、
错误反射、重复物体和不合理背景结构。
避免豪华样板间、酒店广告、时尚写真、网红探店、
空洞站姿和只为展示衣服的摆拍。
禁止未成年感、露点、裸露敏感部位、内衣特写、透明衣物、夸张解剖比例和低俗挑逗姿势。

执行要求：
1. 将两张输入图分别标记为身份参考和造型参考。
2. 生成2张独立候选，不要用一张候选继续滚动人物身份。
3. 对照身份母图检查眼睛、鼻子、嘴唇、下颌、年龄和发际线。
4. 检查手指、眼睛、牙齿、文字、反射、阴影和背景结构。
5. 选择最好的一张；全部变脸时最多重试2轮。
6. 将候选和提示词保存到 emotion_women/output/persona_jobs/。
7. 不加入 image_pool.txt，不修改文章，不发布公众号。
~~~

## 按文章生成三张图

~~~text
使用 imagegen 技能，为下面的公众号文章生成同一人物的3张配图：

文章：
emotion_women/articles/在这里填写文章文件名.md

唯一身份参考：
emotion_women/images/persona/master/persona_master_glam_v3.png

造型参考：
emotion_women/images/persona/style/persona_style_sensual_v1.jpg

身份母图决定她是谁，造型参考只控制穿搭和成熟性感的气质。
所有图片必须是同一个28岁成年女性，保持脸型、眼睛、鼻子、
嘴唇、下颌、年龄、肤色、发际线和黑色齐肩头发一致。

先阅读文章，再设计：
1. 封面图：人物与文章核心情绪，主体清晰。
2. 冲突图：包含完整生活行为和环境，直接对应故事核心矛盾。
3. 结尾图：人物状态发生变化。

三张图需要改变造型、表情、姿势和背景，但不能改变人物身份。
不得全部使用正脸特写，至少包含一个中景和一个环境镜头。
每个场景都必须有具体行为、接触物件和一处真实使用痕迹，
不能只写“在咖啡馆喝咖啡”“在办公室微笑”等空洞摆拍。

统一使用高度写实的生活摄影风格：真实毛孔、自然碎发、
合理布料褶皱、真实光影、35mm或50mm镜头、不过度磨皮。

每个分镜生成2张候选，逐张对照身份母图检查人物一致性，
同时检查手指、牙齿、眼睛、文字、反射、阴影和背景结构。

将全部候选、分镜说明和提示词保存到：
emotion_women/output/persona_jobs/

只生成本地图片，不加入图池、不修改原文章、不发布公众号。
~~~

## 人工确认后的正式入库

确认图片与身份母图是同一个人后，在 emotion_women 目录执行：

~~~bash
../.venv/bin/python persona_asset_tool.py finalize \
  output/persona_jobs/JOB/finals/SCENE.png \
  --slug YYYYMMDD-topic-scene-role \
  --prompt-file output/persona_jobs/JOB/prompts/SCENE.txt
~~~

正式图片默认保留原分辨率，只输出一张 JPG。输出到：

~~~text
emotion_women/images/persona/scenes/
~~~

生成记录输出到：

~~~text
emotion_women/images/persona/metadata/
~~~

正式入库仍不会自动修改图片池或发布公众号。

## 固定原则

- 标准母图只负责身份，不负责性感造型。
- 造型参考只负责穿搭和气质，不负责脸。
- 每张新图都重新使用标准母图，不能把场景成品当作新母图。
- 日常场景必须有具体动作、生活物件和真实使用痕迹，不能只是换背景摆拍。
- 造型参考的落地窗背景和展示型姿势不能复制到普通文章场景。
- “看不出 AI”无法绝对保证，必须人工放大检查生成瑕疵。
- 不使用来源不明的真人照片作为身份、姿势或场景素材。
