# AI 科技资讯公众号 · 使用说明

一套基于 Claude Code 的全自动 AI 科技资讯生产系统。Python 脚本负责调度和注入动态 Prompt，Claude Code 主进程扮演**主编**搜热点、筛选题、去重、派活，`ai-news-writer` 子智能体扮演**写手**做研究并成稿。

> 想快速上手直接看「一分钟上手」；想搞清原理和发布配置往下翻。

---

## 一分钟上手

```bash
cd media_agent

# 立即生成 1 篇（带详细日志，推荐首次运行用这个观察全过程）
python3 daily_ai_news.py --now --count 1 --verbose
```

跑完后草稿在 `output/` 下，文件名形如 `20260702-2227-anthropic-fable5-global-relaunch.md`。默认**只生成本地草稿，不碰公众号**。

## 运行模式

脚本有两个维度的模式：**触发方式**（立即 / 定时）× **是否发布**（本地草稿 / 推送草稿箱）。

### 1. 立即生成（手动触发）

```bash
# 生成 1 篇
python3 daily_ai_news.py --now --count 1

# 生成 3 篇，带详细日志
python3 daily_ai_news.py --now --count 3 --verbose
```

`--now` 表示立即执行一次就退出，不进入定时循环。

### 2. 定时调度（后台常驻）

```bash
# 每天北京时间 08:00 和 20:00 各生成 3 篇
python3 daily_ai_news.py --time 08:00 20:00 --count 3

# 后台守护运行，日志重定向到文件
nohup python3 daily_ai_news.py --time 09:00 --count 5 > run.log 2>&1 &
```

- `--time` 可跟多个时间点，格式必须是 `HH:MM`（24 小时制，北京时间）。
- 定时模式是前台阻塞循环（每 30 秒轮询一次），按 `Ctrl+C` 退出。
- **注意**：`--time` 依赖 `schedule` 库，未安装会直接报错退出。

> 若同时传了 `--now` 和 `--time`，脚本会先立即跑一次，再进入定时循环。

### 3. 发布到公众号草稿箱

```bash
python3 daily_ai_news.py --now --count 3 --publish
```

加上 `--publish` 后，成稿会经 `wenyan-mcp` 排版并推送到微信公众号**草稿箱**（不直接群发，需人工在后台审核后再发布）。配置见下方「发布到公众号」章节。

## 命令行参数

| 参数 | 说明 |
| --- | --- |
| `--now` | 立即执行一次，不等待定时调度 |
| `--count N` | 每次执行生成 N 篇文章（默认 1，必须 ≥ 1） |
| `--time HH:MM [HH:MM ...]` | 设定每日执行的北京时间，可多个 |
| `-v` / `--verbose` | 输出详细日志（`--output-format stream-json`），能看到主编和写手的完整调度过程 |
| `--publish` | 成稿后通过 `wenyan-mcp` 发布到微信公众号草稿箱（需先配好 `.mcp.json`） |

## 工作流程

```
Python 调度层
    └─► Claude Code 主进程（主编 / Agent Manager）
            ├─ 1. WebSearch 搜最近 24h 的 AI/科技热点
            ├─ 2. 筛选选题 + 去重
            ├─ 3. Task 工具分派 ai-news-writer 子智能体（一次一个选题）
            │        └─► 写手：深度研究 → 写 Markdown → 保存到 output/
            ├─ 4.（--publish 时）调 wenyan-mcp 推送到草稿箱
            └─ 5. 向你汇报每篇标题、路径、摘要
```

1. **调度触发** — 按 `--now` 或 `--time` 启动。
2. **动态 Prompt** — 脚本注入当前北京时间、最近 24h 新闻窗口、已发文章标题（用于去重）。
3. **主编执行** — 搜热点 → 筛选 → 用 `Task` 分派写手。
4. **写手撰写** — 多源研究 → 按风格规范写成 Markdown → 存到 `output/`。
5. **汇总** — 主编报告每篇标题、路径、一句话摘要（发布模式还会带草稿箱 media_id 或失败原因）。

## 输出与去重

- 文章路径：`output/YYYYMMDD-HHMM-<英文短slug>.md`（slug 小写英文加连字符，如 `openai-gpt5-release`）。
- frontmatter 只含 `title` 和 `cover` 两个字段。
- **去重双保险**：
  1. 脚本每次运行会扫描 `output/` 下最近 40 篇 `.md` 的 `title:` 行，注入 Prompt 让主编避开重复选题（见 `daily_ai_news.py` 的 `list_existing_titles`）。
  2. 主编还会读写 `output/published.log`，每篇成稿追加一行「日期 + 标题」记录（见 `.claude/CLAUDE.md`）。

`output/` 和 `*.log` 已在 `.gitignore` 中忽略，不会进版本库。

## 发布到公众号（可选）

默认 `output/` 只是本地草稿。要真正推送到微信公众号草稿箱：

1. **构建 [wenyan-mcp](https://github.com/floodsung/wenyan-mcp)**：
   ```bash
   git clone https://github.com/floodsung/wenyan-mcp.git
   cd wenyan-mcp && npm install && npm run build
   ```
2. **配置 `.mcp.json`**：把 `wenyan-mcp` 的 `args` 指向 `dist/index.js` 的实际绝对路径，填入微信公众号 `WECHAT_APP_ID` / `WECHAT_APP_SECRET`（微信公众平台 → 开发 → 基本配置）。
3. **运行时加 `--publish`**：脚本会自动放开 `publish_article` / `list_themes` 两个工具，并以 `--mcp-config .mcp.json --strict-mcp-config` 加载配置。**无需手动改脚本里的工具列表**——`--publish` 已经处理好了。

发布默认进**草稿箱**，人工审核后才正式群发，安全可控。

> ⚠️ `.mcp.json` 含微信 AppID/AppSecret，已被 `.gitignore` 忽略，**切勿提交到版本库**。

### 切换排版主题

排版主题由 `daily_ai_news.py` 顶部的常量 `PUBLISH_THEME` 控制，默认 `lapis`。这个值会被注入主编 Prompt，主编再传给 wenyan-mcp 的 `publish_article` 工具。换主题只改这一行即可：

```python
PUBLISH_THEME = "lapis"   # 改成下表任一 theme_id
```

wenyan-mcp 内置主题：

| theme_id | 风格 |
| --- | --- |
| `default` | 简洁默认 |
| `lapis` | 蓝色系，清爽（当前默认） |
| `orangeheart` | 橙色系，活泼 |
| `rainbow` | 彩虹标题 |
| `pie` | 图文清新 |
| `maize` | 玉米黄，温暖 |
| `purple` | 紫色系 |
| `phycat` | 物理猫，技术风 |

想确认本地这版 wenyan-mcp 实际支持哪些主题，可在发布模式下让主编调用 `list_themes` 工具列出（该工具已随 `--publish` 自动放开）。改完 `PUBLISH_THEME` 无需重启任何服务，下次运行 `--publish` 即生效。

## 设计要点

- **反 AI 检测**：写手规范禁用过渡词堆砌和评价性套话，要求高数据密度、句式长短交错，详见 `.claude/agents/ai-news-writer.md`。
- **不编造**：Prompt 与写手规范均强制事实来自可查证信源，查不到的数据宁可不写。
- **可扩展**：复制 `media_agent/` 目录，替换 `.claude/agents/` 写手风格和 `.mcp.json` 凭证，即可派生其他垂直领域公众号。

## 常见问题

**Q：跑完提示成功，但文章缺数据 / 信源没抓全？**
A：Claude CLI 的 WebFetch/WebSearch 可能被网络策略拦截（日志里会出现 `Unable to verify if domain ... is safe to fetch`）。此时写手会严格基于 WebSearch 摘要写作、不编硬数据，但**正式发布前应在网络通畅时对一手信源再交叉核对一遍**。

**Q：`--time` 报错说缺 `schedule`？**
A：`pip install schedule` 后重试。

**Q：`--verbose` 日志太长看不过来？**
A：`--verbose` 输出的是 stream-json 全量事件（含大量 `thinking_tokens`），调试时重定向到文件再筛关键行更实用；日常运行去掉 `-v` 即可。

**Q：找不到 `claude` 命令？**
A：先安装并登录 Claude Code CLI，确认 `claude --version` 可用。
